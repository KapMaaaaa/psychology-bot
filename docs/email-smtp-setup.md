# Настройка отправки email (SMTP / Gmail)

Бэкенд отправляет коды подтверждения регистрации через **fastapi-mail** по SMTP. Ниже — как подготовить почту и переменные окружения.

## Выбор подхода

- **Gmail + пароль приложения (App Password)** — быстрый старт без сторонних сервисов, подходит для небольшого числа писем (коды регистрации).
- **Транзакционный провайдер** (SendGrid, Resend, AWS SES, Mailgun и т.д.) — удобнее для продакшена, высокой нагрузки и своего домена в поле «От кого». Тот же код приложения: меняются только `MAIL_SERVER`, порт и учётные данные в `.env`.

## Аккаунт Google

1. Используйте существующий Google-аккаунт или создайте отдельный для приложения (так проще не смешивать личную почту и рассылки).
2. Для продакшена разумно иметь **выделенный ящик** только под сервисы.

## Двухфакторная аутентификация (2FA)

Для обычного Gmail **пароли приложений** доступны только если включена **двухэтапная аутентификация**:

1. Откройте [Google Аккаунт](https://myaccount.google.com/) → **Безопасность**.
2. Включите **Двухэтапная аутентификация** (подтвердите телефон или приложение-аутентификатор).

Подробнее: [Google — пароли приложений](https://support.google.com/accounts/answer/185833).

## Пароль приложения (App Password)

1. **Google Аккаунт** → **Безопасность** → **Пароли приложений** (может потребоваться снова ввести пароль).
2. Создайте пароль для приложения (например, имя «Психология-бот»).
3. Скопируйте **16 символов** (буквы без пробелов или с пробелами — оба варианта допустимы).
4. Вставьте значение в **`MAIL_PASSWORD`** в `.env` (не коммитьте файл в репозиторий).

## Соответствие переменным `.env`

| Переменная | Что указать |
|------------|-------------|
| `MAIL_USERNAME` | Полный адрес Gmail, с которого идёт отправка (например `you@gmail.com`). |
| `MAIL_PASSWORD` | Пароль приложения из предыдущего шага. |
| `MAIL_FROM` | Адрес отправителя; для Gmail обычно **тот же**, что и `MAIL_USERNAME`, иначе доставляемость может ухудшиться. |
| `MAIL_FROM_NAME` | Необязательно: отображаемое имя («Be Heard» и т.п.). |
| `MAIL_SERVER` | Для Gmail: `smtp.gmail.com`. |
| `MAIL_PORT` | `587` (STARTTLS) или `465` (SSL). |
| `MAIL_STARTTLS` | Для порта `587`: `true`. |
| `MAIL_SSL_TLS` | Для порта `587`: `false`. Для `465` — обычно `true` и `MAIL_STARTTLS=false` (см. `.env.example`). |
| `MAIL_SUPPRESS_SEND` | `1` — не слать письма, только логировать (локальная разработка без SMTP). |

Все переменные перечислены в **`.env.example`** в корне репозитория.

## Особенности Gmail

- Лимиты на число писем в сутки зависят от типа аккаунта; для массовых рассылок нужен другой провайдер.
- **Свой домен** в поле «От кого» без привязки к Google Workspace и DNS обычно **требует** настройки у почтового/транзакционного провайдера.

## Проверка

1. Заполните `.env`, перезапустите бэкенд (например, `docker compose up` или `uvicorn`).
2. В приложении начните регистрацию и запросите код подтверждения.
3. Проверьте **входящие** и папку **Спам**.

Если письма не приходят, проверьте логи сервера на ошибки SMTP и корректность `MAIL_STARTTLS` / `MAIL_SSL_TLS` для выбранного порта.
